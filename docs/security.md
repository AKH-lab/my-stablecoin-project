# ملاحظات امنیتی

## هشدارهای مهم
- هرگز کلید خصوصی خود را در کد ذخیره نکنید.
- از فایل .env برای مدیریت اطلاعات حساس استفاده کنید.
- فایل .env را در .gitignore قرار دهید.

## بهترین روش‌ها
1. از کتابخانه‌های امن مانند OpenZeppelin استفاده کنید.
2. قراردادها را قبل از استقرار اصلی روی تستنت آزمایش کنید.
3. از ممیزی امنیتی استفاده کنید.

## اقدامات اضطراری
در صورت کشف آسیب‌پذیری:
1. عملکرد قرارداد را متوقف کنید.
2. جامعه را مطلع کنید.
3. وصله امنیتی منتشر کنید.
